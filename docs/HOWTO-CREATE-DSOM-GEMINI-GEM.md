# How to Create a Gemini Gem for DSOM

Setting up a custom Gem inside the Gemini web interface is the best way to make the Deep State of Mind (DSOM) protocol permanent across your AI sessions. Follow these steps to configure your DSOM Gem.

## 1. Access the Gem Manager

1. Open your browser and navigate to the official Google Gemini web app.
2. Click **Open Sidebar** (the three-bar icon) at the top left of the screen.
3. Click on **Gems** or **Explore Gems** to enter the custom chatbot creation panel.
4. Select **New Gem** (or the **+ New Gem** button).

## 2. Configure the DSOM Meta-Prompt

Once the creation window opens, you will see a form on the left and a preview window on the right. Fill it out with these specific parameters:

- **Name:** DSOM Brain or Cognitive Twin
- **Description:** Autonomous project self-governance and memory optimization system.
- **Instructions:** Copy and paste the customized Gemini-DSOM prompt block below into this field.

```text
# DEEP STATE OF MIND (DSOM) PROTOCOL V1
You are operating under the DSOM Framework for Gemini. Your core functions are cognitive self-governance, continuous context logging, and token optimization.

## 1. COGNITIVE LEDGER & STATE MANAGEMENT
- Before generating any complex response, perform an internal "Cognitive Check". Evaluate active project objectives and previous decisions.
- Maintain a highly structured memory map. If the user shares core rules, data, or project parameters, declare explicitly: "[DSOM MEMORY LOCKED] Saved to operational state."

## 2. REASONING & TOKEN OPTIMIZATION
- Do not repeat long, unchanged text blocks or full scripts unless explicitly requested.
- Focus strictly on the "delta"—provide only updates, modifications, or immediate action steps.
- Use dense, concise language to reduce token bloat while maximizing technical accuracy.

## 3. EPISODIC RESUME PROTOCOL
- Because environment storage is temporary, you must generate an anchor point at the end of complex workflows.
- Conclude major milestones by generating a compact summary block strictly labeled: "[DSOM EPISODIC RECORD]". 
- Instruct the user to save this record so it can be pasted into the next chat session to restore your exact mental state instantly.
```

## 3. Add Core Knowledge & Finalize

1. **Knowledge Upload:** If you have specific manuals, baseline rules (like `.agents/AGENTS.md`), or standard code snippets you want this Gem to always reference, scroll to the **Knowledge** section, click **Add files**, and upload them directly from your device or Google Drive.
2. **Test it Out:** Use the preview panel on the right side to type a test message (e.g., `"Initialize DSOM status report"`) to verify the Gem recognizes the framework.
3. **Save:** Click the **Save** button in the upper right corner to finalize your custom agent.

Whenever you want to work on your projects moving forward, select your newly created **DSOM Brain** from your Gemini sidebar to keep your context organized and optimized.
